TOPIC: GitHub Collaboration
===========================

SUMMARY:
GitHub is a platform for hosting Git repositories and collaborating with others.

KEY CONCEPTS:
- Fork: Personal copy of someone else's repository
- Pull Request: Request to merge your changes into another repository
- Clone: Download a repository to your local machine
- Remote: Link to a repository hosted on GitHub

COMMON WORKFLOW:
1. Fork a repository
2. Clone your fork locally
3. Create a feature branch
4. Make and commit changes
5. Push to your fork
6. Open a Pull Request

BEST PRACTICES:
- Keep pull requests small and focused
- Write descriptive PR titles and descriptions
- Respond to review comments promptly
